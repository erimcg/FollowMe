Change Log

  • BC-FollowMe 1-09
  • 1) Fixed a few GoogleMap bugs and errors.
  • a. GPS retrieving old stored location and marking as start point has now been fixed.
  • b. Battery life was consumed when user pressed device home button with application still open because GPS was still running. Now GPS is paused when user presses home button and resumes upon reopening of application.
  • c. GPS recording will not resume if the user presses the home button on their devices if they have already pressed stop to stop
  • d. Fixed issue where the application would crash if the user pressed the stop button two or more times repeatedly.
  • e. Fixed issue where a recording GPS message was displayed as many times as the user pressed start GPS menu button. Now the message is only displayed one time until the user presses the stop button and represses the start.
  • 2) Action overflow menu button deemed unnecessary and has been removed from the menu bar.
  • BC-FollowMe 1-07
  • 1) Fixed a few GoogleMap bugs and errors.
  • 2) Minor code clean up and performance improvements.
  • 3) GoogleMaps will now draw a blue circle radius around the user's location.
  • 4) Application will check upon initializing GoogleMaps if the device's GPS is turned on.
  • a. If not then an alert dialog will be displayed asking the user to enable GPS.
  • b. If the user hits "settings" then the user will be taken to GPS settings.
  • c. If they hit "cancel" then alert dialog is cleared and nothing happens.
  • 5) Application will display alert dialog to user if their is no clear view of satellites asking user to move to a clear view of the sky.
  • 6) GoogleMaps now tracks the user's location, but does not plot points on map at this time.
  • BC-FollowMe 1-05
  • 1) Database changed so that user login page can query database.
